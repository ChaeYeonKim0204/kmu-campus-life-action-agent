## 1. 프로젝트 개요

국민대학교 재학생이 성적증명서를 업로드하면, AI가 요람(1,106p)을 분석하여 **졸업 가능 여부**, **대체 이수 과목**, **마이크로 디그리 달성 가능성**, **자기소개서용 직무 역량**을 자동으로 진단해주는 서비스입니다.

### 핵심 특징

| 특징 | 설명 |
|------|------|
| **Hybrid RAG** | 구조화 JSON(별표5 정확 수치) + ChromaDB 벡터 검색 병행 |
| **Vision OCR** | 이미지 기반 성적증명서 PDF → GPT-4o Vision으로 자동 파싱 |
| **70개 학과 지원** | 별표5 전체 파싱, 두 단계 검색(토큰 일치 → 부분 문자열) |
| **출처 추적** | 모든 답변에 요람 페이지 번호 + 섹션명 표시 |

```

└──────────┬──────────────────┘
           │ TranscriptInfo
           ▼
┌──────────────────────────────────────────────────┐

│                                                  │

└──────────┬──────────────────────────────────────┘
           │
     ┌─────┴──────┐
     ▼            ▼
구조화 JSON    ChromaDB RAG
```

## 4. 설치 및 실행

### 사전 요구사항
- Python 3.9+
- OpenAI API 키 (GPT-4o 접근 권한 필요)

### Step 1: 의존성 설치

```bash
pip install -r requirements.txt
```

### Step 2: API 키 설정

```bash
cp .env.example .env
# .env 파일에서 OPENAI_API_KEY 값을 실제 키로 교체
```

### Step 3: 별표5 구조화 데이터 추출 (최초 1회, 약 10초)

```bash
python3 0_extract_structured_data.py
```

> `graduation_requirements.json`, `department_search_index.json`, `grade_category_codes.json` 생성됨

### Step 4: 요람 인덱싱 (최초 1회, 약 10~20분, API 비용 약 $0.5~1.0)

```bash
python3 1_build_index.py
```

> `chroma_db/` 폴더와 `index_stats.json` 생성됨 (완료 후 1,056청크)

### Step 5: 앱 실행

```bash
streamlit run 3_app.py
# 포트 충돌 시: streamlit run 3_app.py --server.port 8502
```

브라우저에서 `http://localhost:8501` 접속

## 5. 4대 에이전트 기능

### ① 졸업 진단 (`agent_graduation_check`)

**입력:** 성적증명서 정보 (학과, 이수학점, 과목 목록)  
**동작:**
1. 별표5 JSON에서 해당 학과의 정확한 졸업 요건 조회
2. ChromaDB로 부가 규정(핵심교양 영역별 이수, 졸업인증제 등) 검색
3. GPT-4o가 충족/미충족 항목 분석 및 해결 방안 제시

**출력 예시:**
```
| 구분     | 필요 | 이수 | 충족 여부 |
|----------|------|------|-----------|
| 기초교양 |  13  |  12  |  불충족   |
| 핵심교양 |  15  |   6  |  불충족   |
| 전공     |  48  |  60  |  충족     |
| 총합계   | 130  | 113  |  불충족   |

판정: 졸업 불가 — 총 17학점 부족
```

**부가 기능:** 교학팀 제출용 TXT 보고서 자동 생성 + 다운로드

---

### ② 대체 이수 탐색 (`agent_substitute_courses`)

**입력:** 성적증명서 + 대체가 필요한 과목명  
**동작:** 요람에서 해당 과목과 동일/유사 인정 가능 과목 검색  
**출력 예시 (캡스톤디자인 대체):**
- 제너러티브캡스톤디자인 (3학점)
- 졸업연구I(캡스톤디자인) (3학점)
- 졸업작품I(캡스톤디자인) (3학점)

---

### ③ 마이크로 디그리 발굴 (`agent_micro_degree`)

**입력:** 성적증명서  
**동작:** 이수 과목을 국민대 마이크로디그리/소학위 트랙 요건과 매칭  
**출력:** 트랙별 달성률(%) 테이블 + 추가 필요 과목 목록 + 우선 추천 트랙

**부가 기능:** 졸업 후 체크리스트 생성 (`agent_post_graduation`)
- 4단계 (D-90 → D-30 → D+1 → D+30) 할 일 목록 자동 생성

---

### ④ 직무 역량 번역기 (`agent_career_translator`)

**입력:** 성적증명서 + 희망 직무  
**동작:** 이수 과목명 → 취업 자기소개서에 쓸 수 있는 역량 언어로 변환  
**출력:**
- 상위 10개 직무 역량 키워드 (근거 과목 포함)
- 자기소개서 활용 문장 예시 3개
- 보완이 필요한 역량 분석

## 6. 기술 세부사항

### 성적증명서 PDF 파싱

| PDF 유형 | 처리 방식 |
|----------|-----------|
| 텍스트 레이어 있음 | `pdfplumber` 텍스트 추출 |
| 이미지 기반 (스캔) | `pypdf`로 Raw RGB XObject 추출 → PIL/numpy JPEG 변환 → GPT-4o Vision |

**Vision 프롬프트 핵심 지침:**
- 학번(8자리 숫자)과 증명서번호(`제G26-xxxxx`)를 구분하도록 명시
- 입학일(YYYYMMDD 형식)을 학번으로 오인하지 않도록 명시

### 별표5 파싱 (`0_extract_structured_data.py`)

- 요람 195-197p에서 70개 학과의 졸업 학점 구조 파싱
- `normalize_spaced_korean()`: 낱글자 사이 공백 제거 (`경 영 학` → `경영학`)
- 행 끝 7개 숫자 추출 + 합계 검증 (총합 ∈ {120, 130, 136, 160})

### 학과명 검색 (`find_department`)

```
1단계: _tokenize_dept_name()으로 토큰 분리
        예: "경영정보학부경영정보전공" → ["경영정보학부", "경영정보전공"]
2단계: 검색어 토큰과 인덱스 토큰 교집합 크기로 점수 매기기
3단계: 점수 0이면 부분 문자열(substring) 검색으로 fallback
```

### ChromaDB 인덱싱

| 항목 | 값 |
|------|----|
| 모델 | `text-embedding-3-small` |
| 청크 크기 | 800자 (단락 경계 우선) |
| 총 청크 수 | 1,056개 |
| 스토리지 | `./chroma_db/` (SQLite, 27MB) |
| RAG 검색 수 | 에이전트당 8개 |

### 이수구분 코드 매핑

| 코드 | 분류 |
|------|------|
| V | 기초교양 |
| Y | 핵심교양 |
| Z | 자유교양 |
| D, C, M, X | 전공 |
| F | 일반선택 |

## 8. 알려진 한계 및 주의사항

### 정확도 한계
- **모든 결과는 참고용** — 최종 졸업 승인은 반드시 교학팀 확인 필요
- 마이크로디그리 트랙 매칭은 과목 제목 기반이므로 공식 인정 여부 별도 확인 필요
- 핵심교양 영역별 이수 요건(인문/사회/자연 각 영역) 세부 검증은 미지원

### 데이터 한계
- 요람 기준: **2025학년도판** (`2025국민대학교요람_20250910.pdf`)
- 매년 요람 개정 시 `1_build_index.py` 재실행 필요
- 별표5 외 특수 학과(야간, 계약학과 등) 일부 누락 가능

### 비용 참고
| 작업 | 예상 비용 |
|------|-----------|
| 최초 인덱싱 (1,056 청크 임베딩) | ~$0.50 |

| 에이전트 1회 실행 | ~$0.05~0.15 |
