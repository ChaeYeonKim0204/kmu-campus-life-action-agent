API 키를 받기 전이라 코드 토양만 미리 다졌어요. 7가지를 했어요:

| # | 한 일 | 왜? |

### 1-3. 점심 후: Phase 1 — 키 도착! 실제로 켜보기

**(1) 키 받은 직후, smoke 테스트 1발에서 큰 함정 발견** ⚡

처음 GPT 호출 → 응답이 빈 문자열로 옴 (`empty_openai_response`). 사용한 토큰 보니 reasoning 토큰 128개를 다 쓰고 출력은 0개. **알게 된 점:** GPT-5 mini는 "추론 토큰"을 출력 토큰 한도에 같이 포함시킴. 우리가 잡은 150 토큰이 추론에 다 먹혀서 실제 답이 안 나왔음.

**해결책 2가지:**
- `reasoning_effort="minimal"` 옵션 추가 (추론 안 함, 그냥 답만)
- 토큰 상한 키움 (150 → 400, 900 → 1500)

이 패턴은 우리 expand/rerank/polish 노드에 깊은 추론 필요 없어서 안전.

**(2) 백엔드 8001 띄우고 /health 확인** ✓
```
llm.enabled=true, model="gpt-5-mini", error=null
graduation_center.ready=true, chroma.count=1056
```

**(3) 요람 인덱스 빌드 — 또 한 번 발견** 🐛

`scripts/build_graduation_index.py` 돌렸더니 "파일을 찾을 수 없습니다" 에러. 파일은 분명히 있는데?

**알게 된 점:** 한글 파일명은 **NFC(합쳐진 형태)** 와 **NFD(분해된 형태)** 두 가지로 저장될 수 있음. 스크립트의 파일명은 NFD(`ᄀ+ᅮ+ᆨ` 분해됨), 다운로드한 PDF 파일은 NFC(`국` 합쳐짐) → 시각적으론 같은데 컴퓨터엔 다른 파일.

**해결책:** 스크립트에 NFC/NFD 둘 다 받아들이는 정규화 매칭 추가. 미래에 누가 다른 형태로 받아도 안 깨짐.

빌드 결과: **1106페이지 PDF → 1056개 청크로 벡터화 완료** (28MB Chroma DB). 비용 약 14원 한 번.

## 5. 비전공자 팀원을 위한 용어 풀이 (선택)

> 이 섹션은 위 보고서 읽다가 막힌 단어 있으면 참고.

| 용어 | 풀어쓰면 |
|---|---|
| **commit / push** | 변경한 코드를 저장(commit)하고 GitHub에 올리는(push) 작업. 자동차로 치면 "오늘 한 일을 회사 클라우드에 백업" |
| **regression test (회귀 시험)** | 새 코드 추가했을 때 옛날 기능이 깨졌는지 자동으로 확인하는 시험. 자동 — 사람이 일일이 클릭 안 함 |
| **citation / 마커** | 답변 본문에 `[S1]` 같은 표시 + 아래에 출처 목록. "이 말의 근거는 1번 자료에서 왔다" 같은 의미 |
| **chunk** | 긴 문서를 검색하기 좋게 잘게 쪼갠 조각. 요람 1106페이지 → 1056 chunks |
| **벡터 DB / Chroma** | 글자를 의미별로 분류한 검색용 데이터베이스. "출석인정"이랑 "공결"이 비슷한 의미인 걸 컴퓨터가 알게 해줌 |
| **sanitization (sanitizer)** | 출력에서 민감정보(학번, 전화번호, GPA 등)를 자동으로 가리는 처리 |
| **fallback** | "실패하면 대신 이걸로"라는 의미. 우리는 GPT가 실패해도 결정론적 답변으로 fallback해서 서비스 안 깨짐 |
| **MVP (Minimum Viable Product)** | "최소 동작 가능한 제품". 영문 의미는 좋지만 발표 평가엔 "그래서 운영도 될 만한가?" 의문 발생 → 그래서 "MVP" 표현 다 제거 |
| **schema** | 입력 양식. 예: 졸업 진단 액션을 시작할 때 사용자가 입력해야 하는 항목 목록 |
| **token (토큰)** | GPT가 글자를 쪼개서 처리하는 단위. 한국어 1글자 ≈ 1~2 토큰. 비용 단위 (gpt-5-mini는 출력 100만 토큰에 약 2달러) |
