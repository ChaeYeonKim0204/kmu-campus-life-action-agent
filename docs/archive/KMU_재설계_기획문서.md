## 1. 재설계 배경 & 목표

### 1-1. 왜 다시 만드는가

현재 서비스는 기능 파이프라인(RAG 검색 → 답변 조립 → 인용 생성)은 완성되어 있으나, 실제 학생이 사용하기 어려운 상태로 배포 전 전면 재설계가 필요하다고 판단했습니다.

UI 직접 점검에서 확인된 주요 문제는 다음과 같습니다.

| 구분 | 문제 | 심각도 |
|---|---|---|
| UI | 관리자 패널(Vector 오류, API key 상태 등)이 학생 화면에 노출 | 🔴 즉시 수정 |
| UI | 개인정보 가드 테스트용 버튼("내 학번이랑 성적으로 처리해줘.")이 학생에게 보임 | 🔴 즉시 수정 |
| UI | 백엔드 연결 실패 시 "응답을 생성하지 못했습니다." 한 줄만 노출 | 🔴 즉시 수정 |
| UI | 질문 입력창이 두 곳(채팅 영역 내부 + 최하단)에 중복 존재 | 🟡 레이아웃 수정 |
| UI | 학생 설정(학생 상태, 학기, 관심 항목) → 체크박스 → 예시 버튼 → 채팅 순서로 질문창이 밀림 | 🟡 레이아웃 수정 |
| UI | 사이드바 5개 패널이 초기 화면에서 모두 "질문 후 표시됩니다" 플레이스홀더로만 채워짐 | 🟡 레이아웃 수정 |
| UI | 답변이 8개 섹션 줄글로 한꺼번에 쏟아져 가독성 낮음 | 🟢 중장기 개선 |
| BE | CORS `allow_origins=["*"]` — 프로덕션 환경에서 보안 위협 | 🔴 즉시 수정 |
| BE | `uvicorn --reload` 모드로 운영 중 — 프로덕션 부적합 | 🔴 즉시 수정 |
| BE | 관리자 전용 기능(`/ingest/run`, `/ingest/live-refresh`, `/sources`)에 인증 없음 | 🟡 배포 전 수정 |
| AG | Chroma 벡터 DB 미설치(`No module named 'chromadb'`)로 벡터 검색 비활성화 | 🟡 배포 전 수정 |
| AG | `chunks.jsonl` 46→103개로 증가했으나 실제 커버리지 미검증 | 🟡 배포 전 수정 |

### 1-2. 재설계 목표

- **학생 사용 가능한 UI:** 기술적 내용 없이 학생이 질문부터 신청서 작성까지 막힘 없이 진행
- **안정적인 배포 환경:** Docker 기반 컨테이너, 환경변수 분리, 프로덕션 서버 설정
- **신뢰할 수 있는 답변:** 청크 품질 검증, 분류기 튜닝, 테스트 자동화(CI)

### 1-3. 범위 외 (이번 재설계에서 다루지 않음)

- LLM(Claude API 등) 실제 연결 — 구조는 준비된 상태로 유지, 연결은 다음 단계
- 로그인·인증 체계 (학생 개인 맞춤화)
- 모바일 앱

## 2. 핵심 사용자 여정 정의

> 이 섹션은 FE·BE·AG 세 담당자가 **같은 화면**을 목표로 작업하기 위한 공통 기준선입니다.

### 여정 A — 질문 → 답변 확인

```
[진입] 서비스 첫 화면 접속
  ↓
[입력] 질문 입력창에 질문 작성 (예: "수강신청 기간이 언제야?")
  ↓
[선택·선택사항] 학생 상태 선택 (기본/신입생/재학생/복학생/휴학생/졸업예정)
  ↓
[제출] 질문 버튼 클릭
  ↓
[대기] 로딩 인디케이터 표시 (처리 단계별 텍스트 표시 권장)
  ↓
[결과] 답변 화면 렌더링
    ├─ 요약 카드 (핵심 답변 한 문장 + 근거 출처 [S1])
    ├─ 체크리스트 카드 (해야 할 일 N개)
    ├─ 기한 카드 (일정이 있는 경우)
    ├─ 문의처 카드 (담당 부서 + 연락처)
    └─ 근거 출처 카드 (접기/펼치기)
  ↓
[선택·다음 행동] 추천 액션 버튼 클릭 → 여정 B 진입
```

**화면 요구사항 (FE 기준선)**

- 질문 입력창은 화면 진입 즉시 포커스
- 로딩 중 "이슈 분류 중 → 공식 자료 검색 중 → 답변 생성 중" 단계 텍스트 표시
- 답변 섹션은 카드 UI, 각 카드 접기/펼치기 가능
- 에러 시: "잠시 후 다시 시도해주세요. 급한 문의는 학교 공식 포털을 이용해주세요." + 포털 링크

---

### 여정 B — 신청서 초안 작성

```
[진입] 여정 A 답변 화면의 "다음 행동" 버튼 클릭
        예: "출석인정신청서 작성하기"
  ↓
[슬롯 입력] 필요 항목 순차 표시
    예: "결석 날짜는 언제인가요?" → "사유는 무엇인가요?"
    ※ 학번·주민번호·성적 입력 시 즉시 차단 메시지 표시
  ↓
[생성] 초안 문서 렌더링
    ├─ 출석인정신청서 양식 텍스트
    └─ 사용된 공식 근거 출처 표시
  ↓
[완료] "복사하기" 버튼으로 클립보드 복사
```

**화면 요구사항 (FE 기준선)**

- 슬롯 입력은 한 번에 하나씩 표시 (채팅 형식)
- 개인정보 차단 시 붉은 인라인 경고 메시지 표시
- 완료 화면에 "이 초안은 참고용입니다. 제출 전 담당 부서에 확인하세요." 고정 문구 삽입

## 3. 역할별 작업 명세

> **우선순위:** P1 (배포 블로커) / P2 (배포 전 필수) / P3 (배포 후 개선)

| **설명** | 현재 학생 화면 오른쪽 하단에 노출된 "관리자" 섹션(공식자료 수집, Chroma 상태, GPT 상태 등)을 `/admin` 경로의 별도 페이지로 이동 |

### 3-2. 백엔드 (BE)

|---|---|

## 4. 역할 간 인터페이스 계약

> BE 담당자가 아래 구조를 확정하면 FE 담당자가 연동 작업을 시작할 수 있습니다.
> **계약 확정 기한: 1주차 금요일**

### 4-1. POST /ask 응답 구조 (확정 필요)

```jsonc
{
  "answer": "string",           // 답변 전문 (FE-006 카드 렌더링에 사용)
  "issue_type": "string",       // 분류된 이슈 유형
  "citations": [                // 근거 출처 목록
    {
      "id": "S1",
      "title": "string",
      "url": "string",
      "text": "string"          // 발췌문 (최대 140자)
    }
  ],
  "next_actions": [             // 다음 행동 추천
    {
      "action_id": "string",
      "label": "string",
      "description": "string"
    }
  ],
  "checklist": {
    "tasks": ["string"],
    "required_documents": ["string"],
    "application_paths": ["string"]
  },
  "contacts": [
    {
      "name": "string",
      "label": "string",
      "phone": "string"         // optional
    }
  ],
  "deadline": null | {
    "deadline": "YYYY-MM-DD",
    "description": "string"
  }
}
```

### 4-2. 에러 응답 구조 (BE-006 결과물)

```jsonc
{
  "error_code": "privacy_blocked" | "no_source" | "server_error",
  "message": "string",          // 사용자에게 보여줄 한국어 메시지
  "detail": "string"            // 개발용 디버그 정보 (운영 환경에서는 생략)
}
```

### 4-3. POST /actions/start 응답 구조

```jsonc
{
  "action_id": "string",
  "title": "string",            // 예: "출석인정신청서 작성"
  "slots": [
    {
      "key": "string",
      "question": "string",     // 사용자에게 보여줄 질문
      "required": true
    }
  ]
}
```

### 4-4. POST /actions/continue 응답 구조

```jsonc
// 완료 시
{
  "status": "completed",
  "document": "string",         // 초안 문서 전문
  "checklist": ["string"],
  "citations": []
}

// 추가 입력 필요 시
{
  "status": "pending",
  "next_slot": {
    "key": "string",
    "question": "string"
  }
}

// 차단 시
{
  "status": "blocked",
  "error_code": "privacy_blocked",
  "message": "string"
}
```
